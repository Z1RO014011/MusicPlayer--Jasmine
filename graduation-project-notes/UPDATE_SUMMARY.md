# Jasmine Music Player 更新总结

## 一、本次更新目标

本次更新主要围绕毕业项目后续完善展开，重点解决以下问题：

- 原界面风格与 Spotify 相似度较高，缺少独立视觉识别。
- 应用图标仍有 Spotify 风格残留，不适合作为独立项目展示。
- 播放队列逻辑与用户日常使用音乐软件的习惯存在冲突。
- 需要补充一些成熟音乐软件常见功能，提升项目完整度。
- 需要形成后续毕业设计、论文和答辩可复用的开发记录。

## 四、播放队列底层逻辑修复

之前的播放队列行为存在一个明显问题：用户在搜索结果或独立歌曲中点击播放后，下一首可能仍然跳回旧播放列表。这不符合大多数音乐软件的使用习惯。

本次修复后：

- 点击本地搜索结果歌曲时，会把当前搜索结果列表作为播放上下文。
- 点击在线搜索结果歌曲时，会把当前在线搜索结果列表作为播放上下文。
- 如果用户单独播放一首没有队列上下文的歌曲，会重置为单曲播放队列。
- 点击下一首时，不会再错误跳回旧播放列表。
- 单曲循环、列表循环、顺序播放和随机播放统一走同一个下一首计算逻辑。
- 随机播放现在会真正随机选择下一首，并避免在多首歌曲时连续随机到当前歌曲。
- 歌曲自然播放结束时，会保留当前队列上下文。
- 删除队列中的当前歌曲时，会自动切换到合理的替代歌曲；如果队列为空，则停止播放并清理音频源。

核心新增逻辑：

- `getNextQueueIndex(state)`：统一计算下一首歌曲索引。
- `PLAY_SONG`：区分显式队列播放和独立歌曲播放。
- `NEXT`：使用统一的下一首计算逻辑。
- `removeFromQueue(index)`：处理删除当前歌曲后的播放状态。

涉及文件：

- `src/context/PlayerContext.tsx`
- `src/components/SearchView.tsx`
- `src/components/DiscoverView.tsx`

## 五、新增功能

本次新增了两个常见音乐软件功能，提升播放队列的可控性。

### 1. 下一首播放

用户可以在在线歌曲列表中点击“下一首播放”按钮，将歌曲插入当前播放歌曲之后。

行为说明：

- 如果当前已有正在播放的歌曲，则插入到当前歌曲后面。
- 如果当前没有播放歌曲，则直接把该歌曲设为当前播放歌曲并开始播放。
- 该功能接近网易云音乐、QQ 音乐、Spotify 等音乐软件中的“Play Next”操作。

涉及文件：

- `src/context/PlayerContext.tsx`
- `src/components/DiscoverView.tsx`
- `src/i18n/translations.ts`

### 2. 清空队列

队列页面新增“清空队列”按钮。

行为说明：

- 当队列中歌曲数量大于 1 时显示。
- 清空队列后保留当前正在播放的歌曲。
- 避免用户清空队列时误中断当前播放。

涉及文件：

- `src/context/PlayerContext.tsx`
- `src/components/QueueView.tsx`
- `src/App.css`
- `src/i18n/translations.ts`

## 十一、播放状态记忆功能

新增目标：

- 用户关闭或刷新应用后，再次打开时能够继续之前的播放上下文。

实现内容：

- 使用 `localStorage` 保存轻量播放会话。
- 保存当前歌曲、当前进度、音量、随机播放状态、循环模式、播放队列和队列索引。
- 本地歌曲恢复时重新匹配 IndexedDB 中的歌曲对象，避免旧 blob 地址失效。
- 在线歌曲不持久化过期播放 URL，点击播放时继续通过网易云 API 懒加载真实音频地址。
- 关键状态变化时立即保存，播放进度通过定时快照保存，避免连续 `timeupdate` 导致防抖保存无法落盘。
- 刷新页面后不会强制自动播放，避免浏览器自动播放限制；用户点击播放后可从恢复进度继续。

验证结果：

- 在线歌曲播放后刷新页面，底部播放器成功恢复歌曲信息。
- 播放进度从 `0:02` 恢复。
- 歌曲总时长 `3:05` 正确显示。
- `npm run build` 通过。

## 十二、适合写入论文或答辩的表述

本项目在传统音乐播放器功能的基础上，进一步优化了播放队列管理逻辑和用户交互体验。系统通过统一的播放队列状态管理函数，对顺序播放、随机播放、单曲循环和列表循环等场景进行集中处理，减少了不同播放入口之间的状态冲突。同时，项目对界面视觉风格进行了独立化设计，降低了与现有商业音乐软件的相似度，使应用更适合作为完整的毕业设计作品展示。
